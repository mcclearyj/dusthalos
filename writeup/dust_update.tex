\section{Code flow}\label{sec:code}

As stated above, I have tried to make \texttt{dusthalos} more useable and robust than it was before, interfacing with code using configuration files, eliminating hardcoded numbers and column names, separating runner scripts from class definitions, etc. Relatedly, the code now handles input galaxy catalogs much more reasonably: preprocessing is now handled separately from reddening and correlation calculations (rather than one giant mess to do it all), and any processing steps are recorded and timestamped in the headers of output catalogs. The idea here was to simplify switching in different foreground-background catalogs rather than having to change the dust calculation code each time (!!).   


\subsection{Catalog processing}\label{sec:code:catalogs}
The high-level workflow is: 

 \begin{itemize}
    \item Load in the foreground and background catalogs and their 
    respective HEALpix survey masks.
    \item Apply survey masks to the catalogs, keeping only those galaxies that 
    lie in the intersection of both HEALPix masks. 
    \item If needed, create a random galaxy catalog. 
    \item If needed, join the background redshift catalog to a photometry catalog.
    \item Save to file.
 \end{itemize}

\subsubsection{Mask handling}\label{sec:code:masks}

Survey masks are manipulated with the HpMask class defined in \texttt{hp\_mask.py}.

\subsubsection{Creating random catalogs}\label{sec:code:randoms}

Any analysis that uses two-point correlation functions needs to distinguishing random correlations from the real signal of interest; this is usually accomplished with random catalogs (as you both know better than I). In cases where random catalogs are not available, I developed a script to generate random points using the survey HEALPix mask. 

\subsubsection{Adding photometry to background galaxy catalogs}\label{sec:code:photom}
The redMaGiC catalogs (both real and random) have positions and redshifts, but not magnitudes. To obtain 
colors, I join them with their counterparts in the Y3 GOLD catalog on the 
\texttt{COADD\_OBJECT\_ID} parameter. The random galaxy catalog obviously doesn't 
have a corresponding entry in the GOLD catalog, so instead I match randomly. This might be a bad call, because there is (obviously) a color gradient with redshift (see Figure \ref{fig:colorredshift}). \textbf{This is on my to-do list for randoms.}

\subsubsection{Extinction models}
Made it easier to toggle between dust (extinction) models and number of redshift bins in which to calculate $A_V$. 

Models are generated using this package (extinction)

All extinction models require wavelengths at which to compute the extinction. The Virtual Observatory's Filter Profile Service (a thousand blessings be upon them) provide reference wavelengths for the DECam filter set shown in Table \ref{tab:filters}. I chose to calculate the extinction at the reference wavelength $\lambda_{\rm ref}$ for each filter, as it splits the difference between the other two measures. The \texttt{extinction} extinction models also require values of $R_V$ and/or $A_V$ to be set. The values of $R_V$ in the literature fluctuate significantly from galaxy to galaxy (possibly redshift as well), and different $R_V$s will produce different extinction profiles. In the absence of a compelling reason to do otherwise, I kept the default values of $R_V = 3.1$ (Milky Way-type dust) and $A_V = 1$. 

The resulting extinctions in the DECam bandpasses are reproduced below:

\section{Conference abstract}\label{sec:abstract}

I will be presenting a talk at ``The Physics and Impact of Astrophysical Dust: from Star Formation through Cosmology,'' an Aspen Center for Physics Winter Conference from March 3--8th. 

\textbf{Title}: 
New Evidence for Megaparsec-scale Dust Using Maximum Likelihood Estimation

\textbf{Abstract}: 
One of the more surprising astrophysical discoveries of the last decade has been an anomalous reddening of quasars consistent with dust halos (and associated large-scale structure) at cosmological scales. While this finding of extended dust halos has been confirmed several times, I will present a test of this result from a new angle: the use of maximum-likelihood formalism to construct an "optimal estimator" for reddening of galaxy colors by dust grains. By correlating the presence of a foreground galaxy with the excess (non-cosmological) reddening of a background galaxy, I find fresh evidence for the existence of megaparsec-scale dust, transitioning from the one- to two-halo regime at scales of 100-300 kpc. While the initial analysis used the redMaGiC catalog of luminous red galaxies -- notable for its tight dispersion in color and its well-constrained photometric redshifts -- as a background sample, the result appears robust to multiple foreground-background galaxy catalog combinations. I will present an overview of our findings and outline our plans to search for sub-millimeter emission from the extended dust halos in our galaxy sample.
